Street Smarts VR  /  AFSFC Assessment Beat 07 — The Spoke
Network Layer: Connection
The Spoke
The rail that doesn't exist — and what replaces it
Current State
No Upward Rail
Data generated at the node has no pathway to AFSFC. It passes through MAJCOM interpretation layers that filter, summarize, and delay it into uselessness.
No Downward Rail
When AFSFC issues a new standard or requirement, there is no mechanism to deliver it directly to the field. It accumulates at each MAJCOM layer until it becomes noise.
With SSVR
Live Upward Feed
The SSVR platform creates a direct data connection from every node to AFSFC. Raw readiness data — not summaries — flows up continuously. MAJCOM is an observer, not a filter.
One-to-Many Push
AFSFC authors a new standard once. It propagates to all 50 nodes simultaneously via the spoke. No accumulation. No lag. No interpretation loss.
The spoke is the operating system. Both pipes, finally open.

Street Smarts VR  /  AFSFC Assessment Beat 09 — The Plan
From Assessment to Action
So We Built a Plan
Your SMEs identified two distinct problems. The plan addresses both — simultaneously.
Phase 01
Catch Up
Address the accumulated deficit. Build the baseline the force was never given.
Phase 02
Keep Up
Establish the rails that keep the force current as threats, doctrine, and technology evolve.
These are not sequential. Both begin immediately.
↓   Here's what each phase contains
Street Smarts VR  /  AFSFC Assessment Beat 10 — Phase 01
Phase 01: Catch Up
Catch Up
Build the baseline. Close the deficit. Bring the force to a common starting line.
Hardware Endpoint Deployment
Standardized SSVR training systems at all 50 installations. Common hardware, common software, no vendor fragmentation.
C-UAS Training Content
Doctrine-aligned engagement scenarios for current and emerging drone threats. Replaces skeet range as the primary C-UAS qualification pathway.
Instrumented Shotgun Integration
Marksmanship qualification tied to instrumented data. Every engagement scored. Proficiency is verified, not assumed.
Authority to Operate (ATO)
DAF cybersecurity clearance for the SSVR platform. Required before network integration. Process initiated at contract award.
Network Integration
All 50 nodes connected to the hub infrastructure. Data flows up. Standards flow down. The operating system goes live.
Street Smarts VR  /  AFSFC Assessment Beat 11 — Phase 02
Phase 02: Keep Up
Keep Up
Establish the rails. Stay current as threats, doctrine, and technology evolve.
Configuration Review Standards Board
AFSFC-chaired body that reviews and approves training standards updates on a defined cycle. Doctrine evolves — the board ensures the network keeps pace.
Modular Hardware Refresh Program
Rolling upgrades across the stack — HMD, compute, sensors — without full system replacement. The network modernizes continuously, not in expensive cycles.
Continuous Content Updates
Threat-informed scenario updates pushed to all 50 nodes as new C-UAS variants emerge. AFSFC authors once. The field receives immediately.
Readiness Analytics
Ongoing AFSFC visibility into force-wide proficiency trends. Identify gaps before commanders do. Resource training decisions with real data.
Standards Interoperability
As AFFORGEN and DAF policy evolve, the network absorbs updates through the spoke — no reintegration effort, no manual propagation.
